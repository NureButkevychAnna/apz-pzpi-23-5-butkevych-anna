2 ЗАВДАННЯ


В цій лабораторній роботі необхідно внести зміни в документ Vision & Scope, який ви створили минулого семестру, а саме в розділ 3 Scope and Limitations. В цьому розділі необхідно описати: 
3.1 Scope of Initial Release — опис скоупу всього проекту, разом із серверною частиною, мобільним і Web клієнтами та IoT. Описати роботу всієї системи. 
3.2 Scope of Subsequent Releases — продумати як в майбутніх релізах системи можна інтегрувати штучний інтелект. Треба описати 3 ключові аспекти роботи з ШІ: 
* застосування — як ШІ може покращити роботу вашої системи; 
- збирання даних — які дані і яким чином можна збирати для подальшого навчання моделей; 
- навчання/використання — чи треба навчати модель самостійно, донавчати існуючу, чи використовувати готовий API сервісу ШІ. Продумайте як краще за все використати дані які ви можете збирати. 
3.3 Limitations and Exclusions — з описаного функціоналу (з обох попередніх пунктів), які є неочевидні обмеження. Який функціонал міг би бути присутній в системі, але навмисно виключений і чому.

Також було сформовано розділ Limitations and Exclusions, у якому визначено основні обмеження системи, пов’язані з технічними особливостями IoT-пристроїв, передачею даних та використанням ШІ. ДОДАТОК Б
VISION AND SCOPE DOCUMENT (КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ)


1.1 Аналіз проблеми
Рівень радіаційного фону є одним із ключових показників екологічної безпеки та здоров’я населення. Особливо це актуально для регіонів, де існують потенційні джерела радіаційного забруднення — поблизу атомних електростанцій, промислових підприємств, сховищ відходів або зон, що зазнали впливу техногенних аварій. Своєчасний моніторинг і відкритий доступ до інформації про стан радіаційного фону мають важливе значення для прийняття рішень органами влади, екологічними службами та звичайними громадянами.        
На практиці спостерігається обмеженість у доступі до оперативних і точних даних про рівень радіації. Державні системи спостереження зазвичай охоплюють лише певні території, а результати вимірювань часто публікуються із запізненням. Крім того, населення не має можливості самостійно долучатися до збору екологічних даних чи контролювати ситуацію у своєму регіоні в реальному часі.
З огляду на стрімкий розвиток технологій Інтернету речей (IoT), з’явилася можливість створити розподілену мережу недорогих сенсорів, які здатні автоматично вимірювати та передавати показники радіаційного фону на сервер для подальшої обробки і візуалізації. Такий підхід дозволяє сформувати глобальну екосистему громадського моніторингу, де кожен користувач може долучитись до спільного збору даних.
Головна ідея проєкту полягає у створенні доступної, масштабованої системи моніторингу радіації, що поєднує IoT-пристрої для вимірювання показників із веб- та мобільними клієнтами для відображення, аналізу та поширення отриманої інформації.


Мета розробки програмного продукту - забезпечити прозорий, оперативний та зручний інструмент для відстеження радіаційного фону у реальному часі, який сприятиме підвищенню рівня екологічної безпеки, обізнаності населення та розвитку відкритих екологічних даних.


1.2 Аналіз аналогів
На ринку екологічного моніторингу вже існують певні рішення для вимірювання та відстеження рівня радіації, такі як Safecast (Японія), Radiation Network (США) та SaveEcoBot (Україна). Вони надають доступ до карт з радіаційними показниками, зібраними з обмеженої кількості сенсорів або офіційних джерел. Проте більшість таких систем мають низку обмежень:
* користувач не може підключити власний сенсор або IoT-пристрій;
* відсутня інтеграція із сучасними мобільними платформами та сервісами аналітики;
* недостатній рівень персоналізації даних (неможливість отримання сповіщень, фільтрації за власними локаціями тощо);
* відсутність чіткої бізнес-моделі, орієнтованої на різні групи користувачів — від пересічних громадян до підприємств і органів влади.
Запропонований проєкт має на меті усунути ці недоліки шляхом створення відкритої, масштабованої та комерційно стійкої екосистеми для моніторингу радіаційного фону з використанням IoT-технологій.


Переваги запропонованої системи:
ПЗс-1: Масштабованість: можливість підключення необмеженої кількості сенсорів у різних регіонах.
ПЗс-2: Інтегрованість: підтримка мобільних, веб- та серверних платформ із єдиною інфраструктурою даних.
ПЗс-3: Оперативність: відображення показників у режимі реального часу з аналітикою і прогнозами.
ПЗс-4: Доступність: низька вартість IoT-пристроїв дозволяє розгорнути систему навіть на рівні громад або навчальних закладів.
ПЗс-5: Відкритість: можливість публічного доступу до даних через API або відкриту карту.


Модель монетизації
Для забезпечення фінансової стійкості проєкту передбачається комбінована модель монетизації, що включає:
ММ-1: Підписки (Subscription model):
* для приватних користувачів — розширені аналітичні функції, персональні сповіщення, збереження історії даних;
* для організацій — доступ до корпоративної аналітики, статистичних звітів, інструментів прогнозування та експорту даних.
ММ-2: Продаж IoT-пристроїв (Hardware sales):
Виробництво або серійне складання власних сенсорних модулів радіаційного моніторингу з попередньо налаштованим програмним забезпеченням.
ММ-3: B2B-рішення для бізнесу та державних структур:
Надання послуг із встановлення, налаштування та обслуговування мережі сенсорів для промислових підприємств, органів місцевого самоврядування, екологічних інспекцій та лабораторій.
ММ-4: Реклама та партнерства:
Розміщення екологічної або освітньої реклами у мобільному застосунку та на веб-платформі, співпраця з науковими установами та медіа.
ММ-5: Продаж агрегованих даних (Data-as-a-Service):
Надання доступу до знеособлених аналітичних даних для наукових досліджень, стартапів або екологічних організацій.


Цільовий ринок та перспективність
Основними цільовими сегментами ринку є:
РтП-1: громадські організації та волонтерські спільноти, які займаються екологічним моніторингом;
РтП-2: навчальні заклади для освітніх і наукових проєктів;
РтП-3: державні та муніципальні структури, що контролюють стан довкілля;
РтП-4: приватні компанії, які мають підвищені вимоги до екологічної безпеки;
РтП-5: звичайні користувачі, зацікавлені у перевірці безпечності місць проживання, роботи чи відпочинку.


2.1 Опис призначення системи
Програмна система призначена для автоматизованого моніторингу рівня радіаційного фону із використанням технологій Інтернету речей (IoT), веб- та мобільних клієнтів. Її основне завдання — забезпечити збір, зберігання, аналіз і наочне відображення даних про рівень радіації у реальному часі, що надходять із мережі розподілених сенсорів.
Система створює цінність для користувачів завдяки можливості отримання достовірної та оперативної інформації про стан навколишнього середовища. Вона дозволяє виявляти потенційно небезпечні зміни рівня радіаційного фону, здійснювати постійний контроль екологічної ситуації та своєчасно реагувати на можливі загрози.
Для звичайних користувачів система є зручним інструментом перевірки безпечності місць проживання, роботи або відпочинку. Завдяки інтерактивній карті та push-сповіщенням користувач може оперативно дізнаватися про перевищення допустимих рівнів радіації у вибраній локації.
Для організацій, підприємств та наукових установ система відкриває можливості аналізу накопичених даних, побудови графіків змін, формування звітів та прийняття рішень на основі фактичних вимірювань. Державні та муніципальні органи можуть використовувати систему як додаткове джерело екологічного моніторингу для підвищення ефективності управлінських дій у сфері безпеки довкілля.


2.2 Опис бізнес-логіки
MF-1. Відображення карти з рівнем радіації.
Користувач може переглядати інтерактивну карту, на якій показані поточні рівні радіації у різних місцях.


MF-2. Перегляд детальної інформації.
Натиснувши на точку на карті, користувач бачить точні показники сенсора, дату й час вимірювання, а також історію змін.


MF-3. Сповіщення про небезпечний рівень радіації.
Система надсилає попередження (наприклад, через мобільний додаток або email), якщо рівень радіації перевищує безпечний поріг.


MF-4. Перегляд статистики та графіків.
Користувач може переглядати, як змінювався рівень радіації з часом у певній точці або регіоні.


MF-5. Пошук і фільтрація даних.
Є можливість шукати інформацію за містом, датою або конкретним пристроєм, щоб швидко знайти потрібні дані.


MF-6. Особистий кабінет користувача.
Зареєстровані користувачі можуть створювати облікові записи, зберігати улюблені точки моніторингу, налаштовувати сповіщення та мову інтерфейсу.


MF-7. Підключення власних сенсорів.
Користувач або організація може під’єднати власний IoT-пристрій, щоб додавати дані до спільної системи моніторингу.


MF-8. Відкритий доступ до даних.
Дані можна експортувати або переглядати через веб чи мобільний застосунок, що дозволяє ділитися ними з іншими користувачами або організаціями.


MF-9. Мультимовна підтримка.
Інтерфейс системи доступний кількома мовами, що робить її зручною для користувачів з різних регіонів.


MF-10. Офлайн-режим (для мобільної версії).
Мобільний додаток може тимчасово працювати без підключення до Інтернету, зберігаючи останні отримані дані.


3.1 Scope of Initial Release
Початковий реліз системи передбачає реалізацію повнофункціональної екосистеми моніторингу радіаційного фону, що включає серверну частину, IoT-пристрої, веб- та мобільні клієнти.
IoT-пристрої виконують збір даних про рівень радіації в реальному часі. Вони здійснюють вимірювання із заданою періодичністю та передають отримані показники на сервер через мережу Інтернет. У разі втрати зв’язку пристрої здатні тимчасово зберігати дані локально з подальшою передачею після відновлення підключення.
Серверна частина забезпечує прийом, обробку, зберігання та аналіз отриманих даних. Вона виконує перевірку достовірності показників, агрегацію інформації, формування статистики та надання доступу до даних через API. Також сервер відповідає за авторизацію користувачів, управління пристроями та генерацію сповіщень у випадку перевищення допустимих рівнів радіації.
Веб-клієнт надає користувачам доступ до системи через браузер. Основний функціонал включає інтерактивну карту з відображенням рівня радіації, перегляд детальної інформації про сенсори, аналіз історичних даних та формування звітів. Адміністративна частина дозволяє керувати користувачами та пристроями.
Мобільний клієнт забезпечує швидкий доступ до даних із мобільних пристроїв. Користувачі можуть переглядати карту, отримувати push-сповіщення про небезпечні рівні радіації, налаштовувати персональні параметри моніторингу та працювати з останніми збереженими даними в офлайн-режимі.
Таким чином, у межах початкового релізу реалізується базова, але цілісна система, яка забезпечує збір, обробку та візуалізацію даних у режимі реального часу.


3.2 Scope of Subsequent Releases
У наступних релізах передбачається розширення функціональності системи за рахунок інтеграції технологій штучного інтелекту.
* Застосування ШІ.
Штучний інтелект може бути використаний для прогнозування змін радіаційного фону на основі історичних даних, виявлення аномалій у показниках сенсорів та автоматичного визначення потенційно небезпечних зон. Також ШІ може покращити персоналізацію системи, надаючи користувачам рекомендації щодо безпечних маршрутів або зон перебування.
* Збирання даних.
Для навчання моделей можуть використовуватись історичні дані з IoT-сенсорів (рівень радіації, час, геолокація), а також додаткові дані, такі як погодні умови, географічні особливості та інформація з відкритих джерел. Дані збираються автоматично через сенсорну мережу та зберігаються на сервері у структурованому вигляді. Важливо забезпечити очищення даних, фільтрацію шумів і валідацію показників перед використанням у моделях.
* Навчання та використання моделей.
На початкових етапах доцільно використовувати готові API-сервіси штучного інтелекту для аналізу даних і прогнозування. У подальшому можливе донавчання існуючих моделей або створення власних моделей машинного навчання, адаптованих до специфіки зібраних даних. Використання власних моделей дозволить підвищити точність прогнозів і зменшити залежність від сторонніх сервісів.


3.3 Limitations and Exclusions
Попри широкий функціонал системи, існують певні обмеження та свідомо виключені можливості.
До основних обмежень належать залежність від якості та точності IoT-сенсорів, можливі затримки у передачі даних через нестабільне інтернет-з’єднання, а також нерівномірність покриття території сенсорами. Крім того, система не гарантує абсолютну точність вимірювань, оскільки показники можуть залежати від умов встановлення пристроїв.
Щодо інтеграції штучного інтелекту, обмеженнями є необхідність накопичення достатнього обсягу якісних даних для навчання моделей, а також обчислювальні ресурси, необхідні для їх обробки.
Свідомо виключеним функціоналом на даному етапі є:
* інтеграція з державними критичними системами моніторингу (через складність доступу та регуляторні обмеження);
* повністю автономна робота IoT-пристроїв без підключення до мережі;
* високоточні професійні вимірювальні прилади (через їх високу вартість);
* розширені функції прогнозування на основі складних ШІ-моделей (перенесено на наступні релізи).


Такі обмеження дозволяють зосередитися на реалізації основного функціоналу системи, забезпечити її стабільність і поступовий розвиток у майбутньому.


4.3 Опис програмних засобів та інструментів
Для розробки системи моніторингу радіації буде використано сучасні технології, що забезпечують надійність, масштабованість і зручність подальшої підтримки.
Серверна частина розроблятиметься з використанням Node.js як основної платформи, оскільки вона дозволяє ефективно обробляти велику кількість запитів у режимі реального часу. Для створення API застосовуватиметься фреймворк Express.js, а для зберігання даних — PostgreSQL, як зручна документно-орієнтована база даних. Передавання даних від IoT-пристроїв здійснюватиметься через протоколи MQTT або HTTPS.
IoT-клієнти базуватимуться на мікроконтролерах, таких як ESP32 або Raspberry Pi, із сенсорами типу Geiger-Müller. Для прошивки використовуватиметься MicroPython або C++ (Arduino SDK).
Веб-клієнт буде реалізовано на основі React.js, що забезпечує швидку роботу інтерфейсу та інтерактивність. Для стилізації використовуватиметься Tailwind CSS, а для візуалізації даних — бібліотека Chart.js або Recharts.
Мобільний клієнт створюватиметься за допомогою React Native, що дозволяє розгортати додаток одночасно на Android та iOS.
Система керування версіями — Git, із використанням платформи GitHub для розробки та контролю змін.
Середовище розробки — Visual Studio Code, як універсальний інструмент для роботи з JavaScript і Node.js.
Для тестування API планується застосовувати Postman, а для розгортання серверної частини — Docker.
